# Write The Brief

Use existing context, decisions, and project language; interview only when a missing answer would change delivery. Preserve decision IDs and publish to the agreed source of truth.

## Process

1. Define the smallest valuable outcome and explicit non-goals.
2. Trace accepted decisions into observable behavior, affected interfaces, and risks.
3. Choose the strongest practical completion evidence.
4. Name the first end-to-end delivery slice.
5. Write and publish the brief.

```md
# Brief: <work>

## Problem
## Goal
## Non-goals
## Decisions
- DEC-001: <trace>
## Outcome and behavior
## Data, systems, and interfaces
## Evidence
## First delivery slice
## Open blockers
```

Finish only when the brief is usable without hidden guesses. Return its location, remaining blockers or material risks, and the next step—normally `slice-the-work`.
