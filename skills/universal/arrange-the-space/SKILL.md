# Arrange The Space

Purpose: make the environment support the work instead of fighting it.

## Rules

- Fit the space to the task and the person.
- Remove friction before adding structure.
- Keep active work, references, outputs, and archive separate.
- Prefer defaults that are easy to reset.
- Separate scratch capture from durable, versioned artifacts.
- Do not over-organize low-stakes work.
- Make the next session easy to start.

## Audit

Understand:

- task or project
- current workspace
- inputs and materials
- tools and constraints
- repeated friction
- desired session shape
- output format
- review or sharing path
- reset routine

## Arrange

Create or recommend zones:

- inbox: unsorted incoming material
- active: current work in progress
- references: durable material to consult
- output: finished or shared artifacts
- versioned record: commits, dated notes, logs, releases, or document history
- decisions: durable choices and rationale
- review: feedback, checks, approvals, or practice notes
- archive: old material kept out of the way

For a physical space, translate zones into places, containers, labels, desk layout, lighting, devices, and reset cues.

For a digital space, translate zones into folders, docs, boards, notebooks, shortcuts, naming rules, templates, and automation only when useful.

## Versioning

Create or use a Git repo when the workspace produces durable text, code, data definitions, prompts, scripts, briefs, logs, drafts, or reusable templates. Use plain folders, notebooks, documents, or boards for raw capture, private reflection, bulky media, and material that does not benefit from diffs or history.

A good workspace makes it obvious what is scratch, what is active, what is ready to commit or publish, and what is archived.

## Domain Notes

For software, include repo, docs, commands, env files, issue tracker, test commands, and review path.

For analytics, include raw data, working notebooks, validated outputs, definitions, refresh notes, and publication path.

For writing or books, include research, outline, drafts, cuts, style notes, references, and feedback.

For game development, include design notes, playable builds, assets, bugs, playtest notes, and release targets.

For language learning, include input sources, vocabulary capture, speaking prompts, correction notes, and review schedule.
