# Arrange The Space

Fit the environment to the task and person. Remove friction before adding structure; keep scratch, active work, durable references, outputs, and archive distinct.

1. Audit the task, current space, inputs, tools, repeated friction, desired session, output/review path, and reset needs.
2. Create only useful zones: inbox, active, references, output, decisions, review, versioned record, archive.
3. Translate zones into physical places or digital folders/docs/boards; automate only repeated friction.
4. Choose simple naming, session-start, session-end, and reset defaults.
5. Use Git only for durable, diff-friendly artifacts.

```md
# Workspace: <purpose>
Layout: <zone → place>
Naming:
Session start:
Session end:
Reset:
Versioning / scratch boundary:
First setup step:
```

Finish with the resulting layout, changes made, versioning decision, reset routine, and obvious next-session action. Do not over-organize low-stakes work.
