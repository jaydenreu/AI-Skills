# Clarify The Aim

Reduce action-changing ambiguity without doing the work. Inspect existing context first; if operating context itself is missing, recommend `orient-the-field`.

Maintain `A1: <unknown> | action impact | recommended default | blocker/defaultable/deferrable`. Ask the highest-impact blocker one question at a time, lead with a recommended answer and one-sentence rationale, then wait. Challenge vague terms and update the ledger after every response.

Cover only choices that can change the outcome, audience, current/target state, constraints, materials or people, quality bar, risk, failure/default behavior, non-goals, evidence, durable artifact, or order of work.

Record durable choices in the agreed place:

```md
### DEC-001 — <name>
Decision:
Reason:
Evidence or assumption:
```

Finish only when blockers are resolved and remaining ambiguity is accepted as defaultable or deferrable. Return decision IDs or location, open items by status, artifact expectations, and the next fitting skill.
