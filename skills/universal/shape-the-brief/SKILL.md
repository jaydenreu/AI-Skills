# Shape The Brief

Synthesize existing context; interview only when a missing answer would change the work. Preserve decision IDs, use domain language, and define evidence before steps.

## Process

1. Identify the source of truth and smallest valuable outcome.
2. Trace decisions into observable experience or process, affected materials/interfaces, risks, and non-goals.
3. Define the durable artifact or release record and strongest practical evidence.
4. Name the first useful end-to-end step.
5. Publish or return the brief.

```md
# Brief: <endeavor>
## Problem
## Goal
## Non-goals
## Decisions
- DEC-001: <trace>
## Outcome and experience
## Materials and interfaces
## Artifact / release record
## Evidence
## First useful step
## Open blockers
```

Finish only when the brief is usable without hidden guesses. Return its location, artifact location, blockers or material risks, and the next fitting skill—normally `carve-the-path` or `carry-the-step`.
