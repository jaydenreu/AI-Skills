# Orient The Field

Establish the smallest durable operating context; do not perform the endeavor itself. Reuse existing places and separate scratch from durable records.

Track unresolved context as `C1: <unknown> | impact | recommended default | blocker/defaultable/deferrable`. Resolve the blocker that most affects the next action.

## Process

1. Locate the aim, source of truth, decision log, materials, work/output surfaces, and vocabulary.
2. Establish constraints and standards, evidence, versioning, cadence, and done rules.
3. Ask only choices that materially affect progress; recommend defaults and record deferrals.
4. Create or update the smallest useful context note:

```md
Aim:
Source of truth:
Decisions:
Materials / work surface:
Standards / constraints:
Evidence:
Versioned record:
Cadence:
Done means:
Next action:
```

Use Git only for durable, diff-friendly work; use the medium that best fits raw capture, private reflection, or bulky artifacts. Finish when the source of truth, decision capture, evidence, done rules, and next action are resolved or explicitly deferred. Return the note location, changed surfaces, and remaining gaps by status.
