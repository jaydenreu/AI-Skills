# Orient The Field

Purpose: establish the operating context before doing the work.

## Rules

- Do not deliver the project work.
- Reuse existing places, files, tools, and habits before creating new ones.
- Keep the setup as small as the endeavor allows.
- Separate durable context from temporary scratch notes.
- Make the next action obvious.

## Map

Find or ask for the current shape of the field:

- aim: what this endeavor is for
- source of truth: where the latest context lives
- decision log: where durable choices are recorded
- materials: docs, datasets, code, notes, references, tools, assets, people
- work surface: folder, repo, notebook, board, app, studio, room, desk
- standards: quality bar, style, constraints, ethics, privacy, governance, review
- evidence: tests, demos, examples, metrics, drafts, recordings, practice logs, sign-off
- versioning: Git repo, folder history, dated notes, document versions, or another durable record
- cadence: session length, review rhythm, deadlines, maintenance
- vocabulary: names, concepts, terms of art
- done rules: what must be true before the work is considered complete

## Decide

Lock in the smallest useful answers:

1. Where the work lives.
2. Where current context lives.
3. Where decisions live.
4. Where outputs live.
5. What evidence proves progress.
6. What artifact, log, or release record should be versioned.
7. What quality rules apply.
8. What to do next.

## Versioning

Decide whether durable artifacts belong in Git, a folder, a notebook, a document system, a board, or a hybrid.

Use Git when the work is text-heavy, decision-heavy, iterative, reusable, collaborative, or benefits from history and review. Do not force Git for raw capture, private journaling, bulky media, or temporary scratch.

Commit the durable artifact, not every passing thought.

## Domain Notes

For software, include repo docs, commands, branches, checks, standards, and review path.

For analytics, include datasets, definitions, metric ownership, refresh cadence, validation, and publication path.

For creative work, include references, drafts, canon, tone, asset libraries, feedback loop, and versioning.

For learning, include target ability, current level, practice materials, feedback source, review cadence, and progress log.

For personal systems, include the physical or digital place, defaults, reminders, friction points, and reset routine.

## Write

Create or update the smallest useful setup note in the agreed source of truth. Prefer an existing file, board, notebook, or document.

Suggested note:

```md
## Field

Aim:
Source of truth:
Decision log:
Materials:
Work surface:
Standards:
Evidence:
Versioning / record:
Cadence:
Done means:
Next action:
```

## Done

Return:

- source of truth
- decision log
- materials and work surface
- evidence, versioning, and done rules
- immediate next action
- changed files, docs, boards, or workspace notes
