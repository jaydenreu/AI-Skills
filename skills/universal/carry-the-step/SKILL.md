# Carry The Step

Purpose: finish one step cleanly, with proof.

## Rules

- Work on one step only.
- Use the agreed source of truth and materials.
- Prefer the smallest useful change.
- Verify the outcome, not just the effort.
- Record evidence, durable artifacts, and any true follow-up.
- Commit or log the durable result when the workspace calls for it and the user has asked for that level of completion.
- Do not expand adjacent work unless the step cannot be completed otherwise.

## Process

1. Read the step, parent brief, decisions, acceptance criteria, non-goals, and evidence expectation.
2. Inspect the current state.
3. State a short plan:
   - outcome to prove
   - evidence to collect
   - materials, files, tools, spaces, or people touched
4. Do the smallest useful change or action.
5. Check the result with the strongest practical evidence.
6. Refine while keeping the step focused.
7. Verify every acceptance criterion.
8. Record what changed, what proved it, and where the result lives.
9. If appropriate, commit or otherwise version only the durable artifacts for this step.

## Evidence Examples

- software: passing test, build, screenshot, demo, log, review
- analytics: reconciled query, chart, metric explanation, validation note, stakeholder check
- creative: draft excerpt, outline, storyboard, playable prototype, recorded read-through, feedback
- learning: completed exercise, recall score, speaking sample, quiz result, corrected mistakes, practice log
- workspace: before/after inventory, folder tree, checklist, routine trial, reduced friction
- personal: observed behavior, completed session, tracked habit, decision, calendar block

## If Blocked

State the blocker, the attempted evidence, and the smallest next action. Use the tightest reliable proxy check available; do not skip verification silently.

## Done

Return:

- delivered outcome
- evidence and result
- durable artifact, log, publish, or commit status
- acceptance criteria coverage
- completion reference or changed location
- true follow-up work only
