# Routing evaluations

Use only when maintaining `model-route`. Produce the route for each fixture without executing it; pass only when every invariant holds.

1. Exact heading rename → T0, non-material, Luna/low, root writer, no child/reviewer, exact check.
2. Settled bounded endpoint → T1, Terra/medium, optional one worker, proportionate tests.
3. Conflicting governed metric → T3/material, Sol, one writer, source-of-truth blocker, independent review.
4. Tightly coupled cross-system failure → T4/material, Sol/Max root sole writer, independent review.
5. Four independent audit dimensions → T5, routed read-only streams, wait and synthesize, no writer.
6. Coupled database/API/UI definition → one writer after parallel discovery; no competing semantics.
7. Disjoint package slices → T5 with isolated writers, integration owner, combined validation/review.
8. Stale price-dependent route → verify official facts before recommending.
9. Hidden runtime model → separate intended from visible; do not infer.
10. Existing repository routing owner → reuse it; do not create competing policy.
